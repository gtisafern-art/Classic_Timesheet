<h1 align="center">Classic Timesheet</h1>

<p align="center">

## Возможности

- **Табель** — ежедневный ввод часов по сотрудникам с привязкой к ресторану, должности и дате
- **Совмещение / Замещение** — учёт работы сотрудников на чужих должностях
- **Отчёт Т-12** — сводная таблица за период с экспортом в Excel
- **Валидация** — автоматический контроль лимитов рабочих часов по подразделениям и дням недели
- **Расширенный режим** — разблокировка произвольных дат и снятие ограничений (защищён паролем)
- **Корректировки** — возможность исправить ранее сохранённые записи

## Установка

### Требования

- Python 3.10+
- ODBC-драйвер для SQL Server ([ODBC Driver 17+](https://learn.microsoft.com/en-us/sql/connect/odbc/download-odbc-driver-for-sql-server))
- Доступ к MSSQL-серверу с развёрнутой схемой БД

### Шаги

```bash
git clone https://github.com/gtisafern-art/Classic_Timesheet.git
cd Classic_Timesheet

python -m venv venv
venv\Scripts\activate

pip install -r requirements.txt
```

### Конфигурация

Скопируйте `.env.example` → `.env` и заполните параметры подключения:

```ini
MSSQL_SERVER=your-server-name
MSSQL_DATABASE=your-database
MSSQL_USER=your-username
MSSQL_PASSWORD=your-password
USE_WINDOWS_AUTH=True
APP_PASSWORD=your-app-password
```

| Параметр | Описание |
|----------|----------|
| `MSSQL_SERVER` | Адрес SQL Server |
| `MSSQL_DATABASE` | Имя базы данных |
| `MSSQL_USER` | Логин (для SQL-аутентификации) |
| `MSSQL_PASSWORD` | Пароль (для SQL-аутентификации) |
| `USE_WINDOWS_AUTH` | `True` — Windows-аутентификация, `False` — SQL-аутентификация |
| `APP_PASSWORD` | Пароль для расширенного режима |

### Запуск

```bash
python main.py
```

## Развёртывание базы данных

### Вариант 1: Docker (рекомендуется)

Одна команда — поднимает MSSQL 2022 и заполняет тестовыми данными:

```bash
docker compose up -d
```

Затем скопируйте готовый конфиг:

```bash
cp .env.docker .env
python main.py
```

> Контейнер `mssql-init` автоматически выполнит `schema.sql` + `seed.sql` и завершится.
> Данные сохраняются в Docker volume `mssql-data`.

### Вариант 2: свой SQL Server

Выполните скрипты вручную через `sqlcmd` или SSMS:

```bash
sqlcmd -S your-server -U sa -P your-password -i database/schema.sql
sqlcmd -S your-server -U sa -P your-password -i database/seed.sql
```

### Тестовые данные

| Сущность | Кол-во | Примечание |
|----------|--------|------------|
| Подразделения | 8 | 3 с лимитами (группа 1), 1 с лимитами (группа 2), 4 без ограничений |
| Должности | 10 | Повар, официант, бармен, администратор и др. |
| Сотрудники | 25 | 1 уволенный, 2 с исключениями (ЕЖД / БД) |
| Записи табеля | ~55 | Июнь 2026: основные, замещения, совмещения, отпуска, больничные, корректировки |

## Бизнес-логика

### Виды работы

| Вид | Описание |
|-----|----------|
| **Основной** | Стандартная смена сотрудника на своей должности |
| **Замещение** | Сотрудник работает вместо другого на его должности |
| **Совмещение** | Сотрудник выполняет обязанности по дополнительной должности параллельно со своей |

### Статусы дня

| Код | Значение |
|-----|----------|
| `число` | Отработанные часы (например `8`, `10,5`) |
| `ОТ` | Отпуск |
| `Б` | Больничный |
| `б/с` | Отпуск без сохранения заработной платы |

### Лимиты рабочих часов

Для определённых подразделений действуют ограничения максимального количества часов в день. Лимиты зависят от дня недели:

**Группа 1:**

| День | Лимит |
|------|-------|
| Вс, Пн, Вт, Ср, Чт | 12 ч |
| Пт, Сб | 14 ч |

**Группа 2:**

| День | Лимит |
|------|-------|
| Пн, Вт, Ср, Чт | 12 ч |
| Пт, Вс | 14 ч |
| Сб | 16 ч |

### Правила валидации

- **Основные + замещение** — сумма часов за день не должна превышать дневной лимит
- **Совмещение** — проверяется отдельно, часы совмещения также не должны превышать лимит
- **Исключения** — сотрудники с определёнными статусами (ЕЖД, БД) освобождены от лимитов
- **Расширенный режим** — валидация отключается, доступен ввод произвольных дат

### Режимы работы

| Режим | Даты | Валидация |
|-------|------|-----------|
| **Базовый** | Сегодня / вчера / позавчера | Включена |
| **Расширенный** | Любая дата (свободный ввод) | Отключена |

Переключение в расширенный режим защищено паролем (`APP_PASSWORD` в `.env`).

### Корректировки

Если запись уже сохранена, она блокируется от редактирования. Для изменения нужно включить флаг **«Корр.»** — это создаёт новую запись-корректировку с актуальными данными. Система при формировании отчёта Т-12 учитывает только последнюю запись по каждой комбинации сотрудник + дата + вид работы.

### Отчёт Т-12

Формирует сводную таблицу по ресторану за выбранный период:

- Строки — сотрудники (с разбивкой по виду работы)
- Столбцы — дни периода
- Ячейки — коды (часы / ОТ / Б / бс)
- Итоги — дни, часы, отпуска, больничные, б/с
- Экспорт в Excel с форматированием и цветовой кодировкой
